Structured document model
Documents represented as JSON/protobuf tree (paragraphs, styles, tables) — NOT binary file chunks. A 100-page Google Doc is one in-memory model (~1-5MB). OT operates on this semantic model.
Server-authoritative OT
Single serialization point per document. Server assigns global sequence numbers. All clients converge to the same state via transform functions. No split-brain — one truth.
One-operation-in-flight
Client buffers while awaiting ACK. Only 1 unacknowledged op at a time. Additional keystrokes composed into a single buffered op. Prevents transform explosion.
WAL + checkpoint
Every op written to Kafka WAL before applying. Periodic checkpoint (serialize document model to S3 every 30s/100 ops). On crash: load last snapshot + replay WAL. Recovery <2s.
Ephemeral presence
Cursor positions and selection state stored in Redis, not persisted. Fire-and-forget over WebSocket. Heartbeat every 30s for session liveness. 5min idle timeout.
Append-only audit trail
Every edit operation logged immutably. Enables: version history, undo/redo, compliance (SOC2, HIPAA), admin investigation. Never UPDATE or DELETE audit rows.
02 Architecture Click components for deep dives
L1 — Client & Edge
Document Editor + API Gateway + CDN
>
📝Document Editor Client
Rich text editor + local OT engine + WebSocket
Renders document model. Captures edits as OT operations. Applies ops locally (optimistic). Sends to server via WebSocket. Buffers while awaiting ACK.
🚪API Gateway
Auth + Rate limit + WebSocket upgrade
OAuth2/JWT validation. WebSocket upgrade for collab sessions. HTTP for doc metadata. Routes to Collaboration Service based on doc_id hash.
⚡CDN
Static assets + initial doc snapshot delivery
Serves editor JS/CSS bundles. Caches initial document snapshots for fast first load. Geographic distribution reduces TTFB.
L2 — Core Services
Collaboration + Document Metadata + Sharing
>
🤝Collaboration Service
PRIMARY — OT engine + WebSocket manager + in-memory doc state
Heart of the system. Maintains in-memory document model per active doc. Receives ops via WebSocket, transforms against concurrent ops, broadcasts to all editors. Single-writer per document via consistent hashing.
📄Document Metadata Service
Doc info, ownership, version tracking
Manages document titles, creation dates, last modified, owner, collaborators list. PostgreSQL sharded by owner_id. Separate from Collaboration Service for clean separation of concerns.
👥Sharing & Permission Service
ACL + link sharing + access control
Permission check on WebSocket connect (before joining collab session). Roles: OWNER > EDITOR > COMMENTER > VIEWER. Link sharing with optional restrictions.
L3 — Storage Layer
S3 (Snapshots) + PostgreSQL (ops) + Redis (sessions) + Kafka (WAL)
>
🗄Object Store (S3)
Document snapshots as serialized blobs
Stores checkpointed document model every 30s/100 ops. Key: {doc_id}/{revision}. Used for crash recovery + initial load. Old snapshots GC'd after N retained.
💾PostgreSQL
doc_snapshots metadata + doc_ops log
doc_snapshots: tracks {doc_id, revision, blob_url} for each checkpoint. doc_ops: append-only log of every operation per document. Partitioned by doc_id + month.
⚡Redis
Active sessions + cursor positions + presence
Tracks who is editing which document. Stores cursor positions (fire-and-forget). Session heartbeat tracking. Pub/Sub for presence updates across server instances.
📨Kafka
WAL — every op written before processing
Write-ahead log for crash durability. Every op written to Kafka before applying to in-memory model. Partitioned by doc_id. Enables replay on crash recovery.
L4 — Async Layer
Notifications + Audit
>
🔔Notification Service
Comment mentions + share notifications
Kafka consumer on doc-events. Sends email/push for @mentions in comments, document shared, suggestion accepted/rejected. Batches rapid edits.
📜Audit Log Service
Immutable edit history + compliance
Every operation logged to Cassandra. Enables version history ("See edit history"), compliance (SOC2, HIPAA), admin investigation. 7-year retention default.
